DAY 42 SUMMARY — Privacy, Data Responsibilities, Audits & Assessments
1. Privacy Concepts (Video 5.4)
Privacy Implications

Privacy is influenced by:

Laws

Regulations

Social norms

Jurisdiction

National and regional governments define privacy limits and enforcement

Key Privacy Regulations

GDPR (EU)

Strong protections for personal data

Data minimization, consent, right to erasure

HIPAA

Healthcare privacy protections

Privacy Shield / Cross-Border Data Concerns

Controls how data moves between countries

2. Data Subject & Personal Data

Data Subject: The identifiable individual the data belongs to

Personal data includes:

Name

Identifiers

Financial data

Medical data

Individuals may request:

Access

Correction

Deletion (where allowed by law)

3. Data Roles & Responsibilities
Data Owner

Accountable for:

Specific datasets

Governance decisions

Often a senior or executive role

Data Controller

Determines:

Purpose of data processing

How data is used

Data Processor

Processes data on behalf of the controller

Often:

Third-party vendors

Payroll providers

Cloud services

Payroll Example

Payroll department = Data Controller

Payroll company/vendor = Data Processor

4. Data Inventory & Retention

Data Inventory

List of all managed data

Includes:

Owner

Location

Update frequency

Sensitivity

Retention Policies

Define how long data is stored

Must align with laws and regulations

5. Internal vs External Data Use

Internal Use

Collaboration

IT operations

Security checks

External Use

Data shared publicly or with partners

Must comply with applicable laws

Security posture

Laws

Industry regulations

Independent third parties

Logs

Reports

Supporting documentation

Evaluate:

Current activities

Security maturity

Recommendations

Improvement guidance

Consolidated
