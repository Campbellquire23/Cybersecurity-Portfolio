2. Penetration Testing Perspectives
Known Environment

Tester has:

Full knowledge of systems

Network diagrams and configurations

Used to:

Validate controls

Test known defenses

Partially Known Environment

Tester has:

Some information

Common real-world testing scenario

Unknown Environment

Tester has no prior knowledge

Simulates a true external attacker

3. Reconnaissance Phase

Goal: Gather information before exploiting anything

Passive Reconnaissance

No direct interaction with target systems

Examples:

Public websites

Social media

Job postings

Online forums

Harder to detect

Active Reconnaissance

Direct interaction with systems

Examples:

Port scans

DNS queries

OS fingerprinting

Service/version scans

Visible in:

Network traffic

Logs

4. Exploitation Focus

Identify:

Vulnerable systems

Misconfigurations

Exposed services

Reduce attack surface by:

Disabling unused services

Tightening configurations

Phishing Awareness

Many organizations run phishing campaigns

Often:

Automated

Designed to test user behavior

Users may receive:

Immediate feedback after clicking

Recognizing Phishing Indicators

Spelling and grammar errors

Domain inconsistencies

Unexpected attachments

Requests for personal or login information

Urgent or threatening language

Response to Suspicious Messages

Never:

Click suspicious links

Open unknown attachments

Report phishing attempts immediately

Organizations should have:

Clear phishing reporting processes

Monitoring & Metrics

Track:

Click rates

Repeat offenders

Frequency of incidents

First offense:

Training opportunity

Recurring offenses:

Identify patterns

Adjust training strategy
